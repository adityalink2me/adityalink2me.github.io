Prompt It. Play It. Remix It.

Today you are not learning game code. You are becoming a game director: describe a fun idea clearly, let AI build it, play it, and keep prompting until it feels great.

Our Rule Today

You describe the experience. AI handles the code.

You choose

Genre, theme, controls, goal, difficulty, and personality.

AI creates

The complete standalone HTML game.

You play

Test whether it is understandable, exciting, and replayable.

You prompt again

Request focused changes based on what playing revealed.

No JavaScript lesson required. Your skill is directing AI with clear, creative, testable prompts.

A Great Game Prompt Answers Seven Questions
1What game?

Name the genre and core idea.

2What goal?

Explain how players win or score.

3What controls?

Mouse, touch, keyboard, or choices.

4What challenge?

Enemies, time, puzzles, or opponents.

5What feeling?

Fast, funny, tense, clever, or cozy.

6What features?

Score, levels, lives, powers, restart.

7. What format? Ask for one complete standalone HTML file that opens in a browser.

The Prompt That Starts Almost Any Game
Master game promptAct as a creative game designer and expert HTML game developer. Create a fun, polished [GAME GENRE] called "[GAME NAME]" in one complete standalone HTML file. Player experience: - Theme: [THEME] - Goal: [HOW TO WIN OR SCORE] - Controls: [KEYBOARD / MOUSE / TOUCH] - Challenge: [ENEMIES / OPPONENT / PUZZLE / TIMER] - Feeling: [FAST / FUNNY / TENSE / COZY / CLEVER] Include clear instructions, satisfying feedback, score or progress, win/lose states, restart, increasing challenge, and responsive design. Use no external files or libraries. Make it playable immediately when opened in a browser. Before returning the complete HTML, test every control, the ending, and restart.

Your creative job

Replace every bracket with a specific choice. The more clearly you describe the fun, the better AI can build toward it.

Prompt Again | Make It Yours
Creative remix promptThe game works. Keep all current working features, but remix the experience: - Change the theme to [NEW THEME]. - Add one surprising rule: [RULE TWIST]. - Make success feel more satisfying with [FEEDBACK]. - Make the game more replayable with [MODE / RANDOMNESS / SCORE]. - Keep instructions simple and visible. Return the complete updated standalone HTML. Test the new rule, all endings, and restart before returning it.

Rule twists change games

  • Board changes after every round.
  • One power can be used once.
  • Players secretly choose actions.
  • Winning creates a harder rematch.
  • A random event appears occasionally.

When a Game Breaks, Describe the Experience
Repair promptThe attached HTML game has this problem: [DESCRIBE EXACTLY WHAT HAPPENS] I expected: [DESCRIBE WHAT SHOULD HAPPEN] I can reproduce it by: 1. [ACTION] 2. [ACTION] 3. [ACTION] Fix the problem while preserving the current game, controls, design, and working features. Return the complete updated standalone HTML. Test the repaired behavior and nearby features before returning it.

You do not need code vocabulary

Describe what you clicked, pressed, saw, and expected. Good evidence is more useful than guessing the technical cause.

Explain behavior, not codeSlide 19
This replaces technical debugging with user-observation skills.

Add optional simple sound effects generated inside the file with a visible sound toggle. Keep text readable, controls responsive, and animations smooth. Polish should clarify

Hands-on 3 | Three-Prompt Improvement Sprint

Play, choose the biggest weakness, and improve it.

Prompt 1: repair

Fix the most important broken or confusing behavior.

Prompt 2: fun

Add one feature that creates a better decision, challenge, or surprise.

Prompt 3: polish

Improve feedback, clarity, and the replay experience.

Keep backups

Save each working version before replacing it. A new generation can accidentally break something good.

Test after every prompt

Play a complete round before requesting the next change.

Time: 35 minutes
Prompt, play, judge, repeatSlide 23
This is the main iteration block.

Prompt Again | Make It Work Everywhere
Mobile and accessibility promptImprove this game so it is easy to play on both computers and phones. Add clear on-screen touch controls where needed, large tap targets, visible keyboard focus, readable text, strong contrast, instructions that explain every control, optional sound, and a pause option if appropriate. Do not rely only on color to communicate meaning. Preserve the game rules and return the complete updated HTML after testing keyboard, touch-sized layout, win/lose, and restart.

Test quickly

  • Make the browser narrow.
  • Try keyboard only.
  • Check every button label.
  • Confirm sound is optional.
  • Restart several times.
More people can enjoy a game they can operateSlide 25

Responsible Design | Fun Without Tricks
No private data

A simple game should not request names, accounts, contacts, or personal details.

No manipulative loops

Avoid fake rewards, deceptive buttons, and pressure designed only to keep players clicking.

No harmful themes

Review AI-generated stories and visuals for stereotypes or inappropriate content.

Give control

Make sound optional and provide pause, restart, and clear exit points.

Fun respects the player.
