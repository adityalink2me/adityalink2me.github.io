Professional Documents

Some documents must persuade and organize.

A proposal, syllabus, contract note, or MoU has a job: help another person understand the idea, compare options, agree on responsibilities, and take action.

Proposal

Problem, solution, work plan, timeline, cost, and expected value.

Syllabus

Learning goals, sessions, topics, activities, assessment, and resources.

Contract draft

Parties, scope, deliverables, payment, responsibilities, and terms.

MoU

Shared purpose, roles, commitments, duration, review, and signatures.

The output must be clear enough for someone else to useSlide 3

From Idea to Document

Do not ask for a PDF first. Build the document in layers.

1Brief

Explain goal, audience, facts, constraints, and desired decision.

2Outline

Ask for sections before drafting the whole document.

3Draft

Generate editable text with headings, tables, and placeholders.

4Design

Apply colors, spacing, hierarchy, and existing formatting style.

5Export

Move to Word or Docs, then save as DOCX or PDF.

Editable first, polished second, PDF lastSlide 5
Emphasize that PDF is usually a final sharing format, not the best editing format.
Document Prompt Formula

A strong document prompt has 8 parts.

You are helping me create [DOCUMENT TYPE]. Purpose: [WHY THIS DOCUMENT EXISTS] Audience: [WHO WILL READ/APPROVE/SIGN IT] Context: [BACKGROUND FACTS] Content needed: [SECTIONS, DATA, REQUIREMENTS] Tone: [FORMAL, SIMPLE, PERSUASIVE, FRIENDLY] Design: [COLORS, STYLE, BRAND, FORMATTING] Output format: [EDITABLE OUTLINE, TABLES, DOCX-READY TEXT] Before finalizing: ask me [NUMBER] questions if information is missing.

Why it works

  • AI understands the reader.
  • It knows the document type.
  • It follows a structure.
  • It leaves placeholders instead of inventing facts.
  • It prepares text for editing, not just reading.

Proposal Master Prompt

Use this first draft prompt

Create a professional proposal for [PROJECT/IDEA]. Audience: [APPROVER/CLIENT/PARTNER]. Purpose: persuade them to [APPROVE/FUND/PARTNER/SIGN]. Context: [2-5 background facts]. Must include: title, executive summary, problem/need, objectives, proposed activities, deliverables table, timeline table, budget table with placeholders, benefits, risks and mitigation, next steps. Write in clear professional language. Do not invent facts; use [PLACEHOLDER] where information is missing. Make the output editable in Word or Google Docs.

Follow-up prompt

Review this proposal as a strict but helpful reviewer. Find missing information, weak arguments, unclear responsibilities, unrealistic promises, and sections that need evidence. Then rewrite the proposal to be clearer and more convincing while keeping it generic and editable.

Respect Existing Design

Tell AI to follow the design you already use.

If your school, company, NGO, or office already has colors, letterhead, fonts, table style, or report formatting, AI should adapt to that style instead of making a random design.

Design reuse prompt

Analyze the style of the attached/sample document first. Reuse its design rules: colors, heading levels, table style, spacing, tone, logo placement, footer style, and section order. Do not change the brand identity. Create the new proposal in the same visual language, with editable text and tables.

How to Give Design Feedback

Use precise feedback instead of saying "make it nice."

AI responds better when you describe what should change: hierarchy, spacing, color, table readability, page breaks, and tone.

Choose a design problem.

Useful design words

  • Visual hierarchy
  • Consistent headings
  • Readable table
  • White space
  • Brand colors
  • Page break before new section
Clear feedback makes the next draft betterSlide 10
Ask learners to rewrite vague feedback into precise feedback.
Prompt for DOCX-Ready Drafts

Editable document prompt

Format this as a DOCX-ready document that I can paste into Word or Google Docs. Use: - Heading 1 for main sections - Heading 2 for subsections - Bullets only where they improve scanning - Tables for timeline, deliverables, budget, and responsibilities - [PLACEHOLDER] for missing facts - Signature blocks where needed Do not use decorative text that will be hard to edit.

Editable means

  • Real headings, not only bold text
  • Tables with short cell text
  • Plain section labels
  • Clear placeholders
  • No image-only text
  • No locked design that prevents editing
DOCX is for editing and collaborationSlide 11
Show why an image-based proposal is difficult to edit.
Prompt for PDF-Ready Output

PDF is the final presentation layer.

Ask AI to prepare clean page flow before exporting. The actual PDF may be saved from Word, Google Docs, Canva, or another document editor.

Make this proposal PDF-ready. Check for: 1. Clear title page or header 2. Executive summary on page 1 3. Page breaks before major sections 4. Tables that fit landscape or portrait pages 5. Short paragraphs and readable bullets 6. Consistent colors and heading levels 7. Final approval/signature section Give me a page-by-page layout plan before rewriting.

Export habit

  1. Edit in DOCX or Google Docs.
  2. Check page breaks and tables.
  3. Export or download as PDF.
  4. Open the PDF and inspect it.
  5. Fix the editable file, then export again.
PDF should be checked visually before sendingSlide 12
Explain that AI can guide layout, but the user must inspect the final PDF.

Refine the Proposal
Reviewer mode

Ask AI to critique the draft from the reader's point of view.

Act as the approver. What would make you hesitate to approve this proposal?
Evidence mode

Ask what facts, numbers, or examples would make the proposal stronger.

List claims that need evidence and suggest what evidence to add.
Design mode

Ask AI to improve layout without changing the message.

Improve section order, table layout, headings, and PDF page flow.

Rule: Always ask AI to explain what it changed. You need to learn from the improvement, not only receive a better file.

MoU prompt starter

Before Sending Any Document

Use the DOCUMENT checklist.

DDecision

Is the requested decision or action clear?

OOwnership

Are responsibilities and names clear?

CConsistency

Do colors, headings, tables, and terms match?

UUnknowns

Are missing facts marked as placeholders?

MMedium

Is DOCX for editing and PDF for final sharing?

Never send a document you have not opened and checkedSlide 17
The acronym continues with E, N, T verbally: Evidence, Next step, Trusted review.
Finish Strong

Your idea deserves a document that makes it easy to say yes.

Use AI to draft, organize, review, and format. Keep the file editable while improving it. Use existing design and colors when available. Export to PDF only after checking the final layout.
