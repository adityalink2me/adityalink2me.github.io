What Is a Dashboard?
Monitor

Shows current status: sales today, attendance this week, tickets open, campaign spend.

Diagnose

Helps users find what changed and where: by region, channel, product, date, or team.

Decide

Supports an action: continue, stop, investigate, allocate budget, contact someone, improve process.

Dashboard rule: If the viewer does not know what to do next, the dashboard is not finished.

Dashboards are decision tools, not chart collectionsSlide 3

Dashboard Language: Dimensions and Metrics

Dimensions

Dimensions describe the record. They are usually categories, dates, names, locations, products, channels, or statuses.

DateChannelRegionProduct

Metrics

Metrics are numbers that can be counted, summed, averaged, or compared.

LeadsSalesRevenueRating
Charts combine dimensions and metricsSlide 5
If fields are typed incorrectly, charts and filters may behave strangely.
Common Dashboard Types
Executive summary

Few KPIs, key trend, top risks, and next actions. Best for leaders.

Operations tracker

Daily or weekly status, filters, tables, and exceptions. Best for teams.

Marketing dashboard

Channels, spend, leads, conversion, campaign performance, and revenue.

Student or class dashboard

Attendance, assignments, progress, support needs, and improvements over time.

Finance dashboard

Revenue, expenses, budget variance, cash flow, and categories needing review.

Project dashboard

Milestones, tasks, delays, blockers, owners, and risk status.

Hands-on | Connect Google Sheets

Steps

  1. Open https://datastudio.google.com/navigation/reporting
  2. Create a blank report.
  3. Choose Google Sheets as data source.
  4. Select the Day 15 spreadsheet and worksheet.
  5. Confirm first row contains headers.
  6. Add the data to the report.
  7. Check field names and data types.

First checks

  1. Date should be Date.
  2. Revenue, Sales, and Leads should be numeric.
  3. Channel, Region, and Product should be text.
  4. Rating should be numeric or decimal.
  5. No blank header names.
Time: 20 minutes

Build the KPI Row
Total Leads

Scorecard using SUM of Leads. Shows demand or interest.

Total Sales

Scorecard using SUM of Sales. Shows completed outcomes.

Total Revenue

Scorecard using SUM of Revenue. Format as currency if appropriate.

Conversion Rate

Calculated field: Sales / Leads. Format as percent.

Average Rating

Scorecard using AVG of Rating. Useful for quality signal.

Date Range

Control that lets viewers change the reporting period.

Calculated Fields with AI Help
Calculated field promptI am building a Looker Studio dashboard from Google Sheets. Fields: [PASTE FIELD NAMES AND TYPES] I need calculated fields for: [EXAMPLE: conversion rate, revenue per sale, lead quality segment] Write beginner-friendly Looker Studio calculated field formulas. For each formula, explain: - Field name - Formula - Data type or format - What it means - What to check if it returns errors Do not assume fields exist unless I listed them.

Add Interactivity with Controls
Date range control

Lets viewers choose the reporting period. Use when data includes dates.

Drop-down filter

Best for Channel, Region, Product, Team, Class, or Status.

Search box

Useful when there are many names, products, or locations.

Chart interaction

Let viewers click a chart to filter the report if the interaction is clear.

Page filter

Useful when one dashboard has pages for summary, details, and exceptions.

Reset habit

Teach viewers how to clear filters so they do not misread partial views.

Interactivity should answer user questions, not create confusionSlide 13
Too many filters make beginner dashboards harder to use.
Add Insight Text Boxes

Use text to guide attention

  1. State the most important finding.
  2. Name the metric and time period.
  3. Mention a likely limitation.
  4. Suggest what to check next.
  5. Keep it short enough to scan.
Dashboard insight promptHere are my dashboard charts and summary values: [PASTE CHART TITLES + KEY NUMBERS] Write 3 short dashboard insight notes for [AUDIENCE]. Each note should: - Start with the main finding. - Mention the supporting metric. - Avoid claiming a cause unless proven. - Suggest one follow-up action. - Use plain language. Keep each note under 25 words.

Dashboard Design Checklist
Readable

Can a viewer read titles, labels, and numbers without zooming?

Consistent

Are colors, date ranges, metric names, and number formats consistent?

Focused

Does each chart support the main decision?

Interactive

Do filters change the expected charts?

Verified

Do dashboard totals match Google Sheets totals?

Safe

Are sharing permissions appropriate for the data?

AI Prompt | Dashboard Audit
Copy-ready audit promptAct as a dashboard reviewer. Dashboard purpose: [PURPOSE] Audience: [AUDIENCE] Charts and KPIs: [LIST DASHBOARD COMPONENTS] Filters: [LIST FILTERS] Key insights: [PASTE INSIGHT TEXT] Audit the dashboard for: 1. Clarity of purpose. 2. Missing KPIs or unnecessary charts. 3. Better chart type choices. 4. Confusing labels or titles. 5. Data quality and formula risks. 6. Privacy or sharing risks. 7. One high-impact improvement I should make first. Be practical and beginner-friendly.

Use after first draft

  1. List what is on the dashboard.
  2. Ask for critique.
  3. Choose one improvement.
  4. Update the dashboard.
  5. Verify with a classmate.

Share Safely

Before sharing

  1. Remove private rows or sensitive columns.
  2. Check report data credentials.
  3. Use viewer access unless editing is needed.
  4. Test the link in another browser or account.
  5. Confirm filters do not reveal hidden sensitive details.

Share options

  1. Private report for yourself.
  2. Specific people with viewer access.
  3. Class/team share link.
  4. Embedded report on a site, only for public-safe data.
  5. PDF export or screenshot for static reporting.
Never publish sensitive dashboards publicly
Dashboard publishing is a data access decisionSlide 17
This matters because Looker Studio reports can be easy to share widely.
Mini Project | Build and Present a Dashboard

Required components

  1. Dashboard title and purpose.
  2. 3 KPI scorecards.
  3. One trend chart.
  4. Two comparison charts.
  5. One filter control.
  6. One detail table.
  7. Two short insight notes.
Time: 55 minutes

60-second presentation

  1. Who is the dashboard for?
  2. What decision does it support?
  3. What is the most important KPI?
  4. What chart gives the strongest insight?
  5. What limitation should viewers remember?
Present the decision, not every chartSlide 18
Keep presentations short so everyone gets feedback.
