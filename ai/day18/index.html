Core Idea
Repeats a rule

When something happens, the automation follows a prepared instruction.

Moves information

It can copy details between forms, spreadsheets, email, calendars, and task tools.

Starts a next step

It can notify a person, create a task, save a file, or ask for approval.

Simple definition: automation is a workflow that runs without you manually doing every click.

If it is repeated, predictable, and low-risk, it may be a good automation candidateSlide 2
Ask learners to name one repeated task they do every week.
Zapier Explained Simply

Zapier is a connector between apps.

It lets non-programmers create workflows called Zaps. A Zap watches one app for an event, then performs one or more actions in other apps.

No-code builderApp connectorsMulti-step workflowsTesting tools

Example

  1. A parent submits a Google Form.
  2. Zapier detects the new response.
  3. It adds a row to a tracking sheet.
  4. It sends a confirmation email.
  5. It creates a follow-up task for staff.
Zapier turns app events into app actionsSlide 3
Avoid promising free limits or specific pricing. Accounts and plan limits change.
Automation Words You Need
Trigger

The event that starts the workflow, such as a new form response.

Action

What happens next, such as send email, add row, or create task.

Zap

Zapier's name for one automation workflow.

Step

One trigger or action inside a workflow.

Filter

A rule that lets the workflow continue only when conditions are true.

Path

A branch that sends different cases to different actions.

Field

A piece of data, such as name, email, class, amount, or date.

Test

A trial run that checks whether each step behaves correctly.

Learn the words once and every automation tool becomes easierSlide 4

Before You Connect Apps
Private data

Do not send student records, personal IDs, health data, or confidential client details unless policy allows it.

Permissions

Know which account owns the automation and which apps it can read or write.

Human approval

Keep approvals for payments, grades, discipline, hiring, public posts, and sensitive messages.

Audit trail

Keep a simple record of what the automation changes and where errors are logged.

Testing data

Test with sample records before using real names, emails, or official documents.

Owner

Assign a person to maintain, pause, or repair the workflow when something changes.

Example Automation | Form to Sheet to Email

Scenario

A school collects workshop registrations through a Google Form. Staff want a clean tracker and automatic confirmation message.

Trigger: new form responseAction: add rowAction: send email

Fields to map

  1. Name.
  2. Email.
  3. Role or class.
  4. Preferred session.
  5. Question or note.
  6. Submission timestamp.

Make Automations Smarter
Filter

Continue only if the form answer says "urgent" or the email comes from an approved address.

Formatter

Clean dates, names, phone numbers, text, or numbers before sending them onward.

Path

Send high-priority requests to one action and normal requests to another action.

Beginner rule: build the basic version first. Add filters, formatting, and branches only when the simple Zap works.

Complexity should solve a real problem, not impress the builderSlide 14
Do not overbuild during the first class project.
Testing Checklist
Happy path

A normal record moves through every step correctly.

Missing field

The workflow behaves safely when a name, email, date, or category is blank.

Wrong condition

Filters stop records that should not continue.

Duplicate

The workflow does not create repeated tasks or emails by mistake.

Permission

The owner account can still access every connected app.

Recipient check

Messages go only to intended people, with correct subject and body.

Test with sample records before trusting real recordsSlide 15
Have learners create fake test data like Test Student and test@example.com.
When an Automation Fails

Common causes

  1. App password or permission changed.
  2. Column name changed in a spreadsheet.
  3. Field mapped into the wrong place.
  4. Filter condition is too strict.
  5. Tool plan or usage limit was reached.
  6. The app changed its interface or API.

Repair habit

  1. Read the error message.
  2. Find the failing step.
  3. Retest that step with sample data.
  4. Fix one thing at a time.
  5. Run a full end-to-end test.
  6. Write down what changed.
Good automation includes maintenanceSlide 16
Normalize failures as part of connected systems, not personal mistakes.
Create a One-page Automation SOP

Document the workflow

  1. Workflow name and purpose.
  2. Owner and backup owner.
  3. Connected apps and accounts.
  4. Trigger and actions.
  5. Fields being moved.
  6. Risks and review points.
  7. How to pause or repair it.
SOP promptCreate a one-page standard operating procedure for this automation: [PASTE WORKFLOW] Include: - Purpose. - Owner. - Trigger. - Actions. - Data fields. - Privacy notes. - Test checklist. - What to do if it fails. Write it so a new staff member can understand it.
If nobody can explain it, nobody can maintain itSlide 17
The SOP is part of the final output, not optional decoration.
