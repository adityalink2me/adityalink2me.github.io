Steps to Create a Strong Portfolio
1. Prepare public details

Collect your name, role, short bio, skills, projects, contact method, and any other information that can safely be public.

2. Choose the website style

Decide the feel before building: minimal, bold, creative, academic, startup, developer, designer, or professional service.

3. Build with AI tools

Use Lovable, Bolt, v0, Figma Make, or another builder to create the site nicely, then push the slides and website files to GitHub.

Simple rule: the best portfolio is specific, honest, easy to scan, and safe to share.

Step 1 | Prepare Personal Details

Safe to make public

  1. Name you want to use professionally.
  2. Role or learning focus.
  3. Short public bio.
  4. Skills and tools.
  5. Projects you are proud of.
  6. Public email or contact form.
  7. LinkedIn, GitHub, or portfolio links.

Keep private

  1. Home address and exact daily routine.
  2. Phone number unless intentionally public.
  3. Private school, office, or client details.
  4. Personal IDs, documents, and passwords.
  5. Photos or names of others without consent.
  6. Anything you would not want searchable later.
Public means searchable

Step 2 | Choose a Website Style
Minimal professional

Quiet colors, strong typography, lots of whitespace. Best for resumes, consulting, and academic profiles.

Creative showcase

Expressive layout, project visuals, personality, and visual experiments. Best for design, media, and creative work.

Developer portfolio

Project cards, GitHub links, technical stack, demos, and problem-solution-results structure.

Startup builder

Sharp hero, product-thinking language, proof, metrics, testimonials, and call to action.

Student portfolio

Learning journey, class projects, skills in progress, certificates, and reflection.

Service portfolio

Offer, process, past work, pricing signal, trust markers, and inquiry form.

Style should match audience, not just personal tasteSlide 5
Ask learners who they want to impress: employer, client, admissions, collaborator, or teacher.
AI Prompt | Portfolio Style Brief
Copy-ready style promptAct as a portfolio website designer. Audience: [WHO SHOULD VISIT THIS PORTFOLIO] Personal brand: [3 WORDS: e.g., practical, curious, reliable] Portfolio goal: [JOB, CLIENTS, INTERNSHIP, UNIVERSITY, FREELANCE, PERSONAL BRAND] Content I have: [LIST SECTIONS AND PROJECTS] Suggest 3 website style directions. For each, include: 1. Visual mood. 2. Color palette. 3. Font style. 4. Homepage layout. 5. Project card style. 6. What to avoid. 7. Which style fits my goal best and why. Keep the suggestions realistic for a beginner using an AI website builder.

Use before building

  1. Pick audience first.
  2. Choose one style direction.
  3. Do not mix every idea.
  4. Save the chosen palette and layout.
  5. Use the same style across pages.
A portfolio should feel intentional before it feels fancySlide 6

Step 3 | Build with AI Website Tools
Lovable

Good for turning app or website ideas into editable web projects from prompts.

Bolt

Useful for generating and editing web apps in a browser-based coding workspace.

v0

Strong for generating polished UI components and page layouts from prompts.

Figma Make

Useful when you want to explore visual layout and design direction with AI-assisted design.

Tool choice is flexible. The important workflow is: prompt clearly, inspect carefully, edit honestly, and publish responsibly.

Copy-ready website promptCreate a polished personal portfolio website for: Name: [PUBLIC NAME] Role or focus: [ROLE / FIELD / LEARNING FOCUS] Audience: [EMPLOYERS / CLIENTS / TEACHERS / COLLABORATORS] Style: [MINIMAL / CREATIVE / DEVELOPER / STUDENT / SERVICE] Sections: - Hero with name, role, and one-line value statement. - About section with short public bio. - Skills section grouped clearly. - Projects section with 3 project cards. - Contact section with safe contact method. Projects: [PASTE PROJECT TITLES + 1-2 LINE DESCRIPTIONS] Requirements: - Make it responsive for mobile and desktop. - Use clear headings and readable text. - Keep the design clean and professional. - Do not invent achievements, employers, awards, or metrics. - Use placeholder images only if I have not provided public-safe images. - Include obvious calls to action: View Projects and Contact Me.

Write Project Cards That Prove Skill

Project card formula

  1. Project name.
  2. Problem or goal.
  3. Your role.
  4. Tools used.
  5. What changed or what you learned.
  6. Link to demo, code, document, or screenshot.
Project card promptTurn these rough project notes into portfolio project cards. Notes: [PASTE NOTES] For each project, write: - Short project title. - One-sentence problem or goal. - My role. - Tools used. - Result or learning outcome. - One honest limitation or next improvement. Do not exaggerate, invent metrics, or claim I did work that I did not do.

Prompt Again | Improve the First Draft
Clarity pass

Make the hero, headings, navigation, and calls to action understandable in five seconds.

Trust pass

Remove exaggerated claims, add evidence, and make project descriptions specific.

Design pass

Align spacing, color, typography, project cards, and mobile layout.

Revision promptReview this portfolio draft for clarity, trust, and professional polish. Keep the same overall style, but improve the hero section, section order, project cards, mobile readability, and contact call to action. Remove any generic or invented claims. Return a revised version and list the exact changes made.
One focused revision beats ten vague requestsSlide 11

Portfolio Quality Checklist
Public safety

No private addresses, phone numbers, confidential work, or unapproved photos.

Truthfulness

No invented employers, awards, client names, results, or fake testimonials.

Readable design

Strong contrast, clear headings, readable body text, and comfortable spacing.

Mobile ready

Navigation, project cards, buttons, and forms work on a small screen.

Working links

GitHub, LinkedIn, demos, contact links, and downloads open correctly.

Fast scan

A visitor understands who you are, what you do, and what to click next.

Optional Command-line Git Flow
Commandsgit init git add . git commit -m "Add portfolio website" git branch -M main git remote add origin https://github.com/USERNAME/REPO.git git push -u origin main

Before running commands

  1. Replace USERNAME and REPO.
  2. Check you are in the correct folder.
  3. Do not commit private files.
  4. Preview the site locally.
  5. Confirm GitHub shows the files.
Never commit passwords or secrets

Peer Review | 10-minute Portfolio Test

Reviewer tasks

  1. Open the portfolio on desktop.
  2. Open it on a phone or narrow screen.
  3. Say what the person does in one sentence.
  4. Click every important link.
  5. Find one unclear claim.
  6. Suggest one improvement.

Creator listens for

  1. What did they notice first?
  2. Where did they get confused?
  3. Which project felt strongest?
  4. What seemed generic?
  5. What should be fixed before sharing?
Time: 10 minutes
A portfolio is for visitors, so visitors should test itSlide 15
Creators should not defend the site during review. They should listen and write fixes.
